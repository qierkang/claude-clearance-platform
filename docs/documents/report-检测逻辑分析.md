# Claude 检测逻辑分析

## 本地检测当前逻辑

当前项目是一个 Astro 单页工具，核心检测在 `src/config/signals.ts` 与 `src/scripts/detect.ts`：

- 浏览器本地检测 6 个信号：系统时区、浏览器语言、中文字体、Intl Locale、时区偏移、Emoji/UA 风格。
- 权重合计 100：时区 30、语言 24、字体 20、Intl 10、时区偏移 8、Emoji 8。
- 每个信号输出 `0..1` 相似度，乘以权重后累计为风险分。
- 风险分档：`0-30` 低，`31-60` 中，`61-100` 高。
- 服务端 `/api/check` 只能读取请求头与部署平台 geo header，因此只估算 IP 时区、国家、Accept-Language、User-Agent，并按可见权重归一化。

关键判断：系统时区是最接近 Claude Code 公开逆向描述的信号；其他信号是环境画像，不能等价为官方判定。

## ipinfo.cv/claude-ai-check 可参考维度

该站点普通请求会触发 Cloudflare challenge，无法直接读取页面内部脚本。公开可验证的页面定位是 Claude AI 可用性/IP 风险检测，可参考的检查维度包括：

- 出口 IP 与国家/地区。
- ASN、机房/住宅网络、代理/VPN 风险。
- WebRTC 泄漏。
- DNS 泄漏。
- 多区域出口与分流一致性。
- 浏览器语言、时区、系统区域设置。

这些维度与本地环境检测互补：前者偏网络出口，后者偏浏览器/系统环境。

## 升级版设计

`Claude 通关局` 保留本地优先的隐私边界，同时把 UI 升级为三层通关矩阵：

- 本地环境：时区、语言、字体、Intl、Emoji/UA。
- 网络出口：出口 IP、ASN、国家、WebRTC、DNS、分流一致性。
- 证据解释：每项信号展示来源、可信度、风险贡献和修复建议。

网络出口类检测不应伪装成本地检测；如果需要真实实现，应明确告知用户会访问服务端或第三方 IP API。
